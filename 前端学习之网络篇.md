# 🌐 前端必备网络与通信知识体系

本清单旨在帮助前端工程师深入理解数据在客户端、网络和服务器之间传输的机制，涵盖 HTTP 协议、网络安全、性能优化等核心内容。

## 阶段一：传输层与应用层协议基础 (Layer 4 & 7)

核心目标：理解数据是如何可靠、高效地在互联网上传输的。

| 知识点 | 核心内容 | 推荐学习资料 |
| :--- | :--- | :--- |
| **TCP/IP 基础** | 理解 **OSI 七层模型**和 **TCP/IP 四层模型**。重点关注**传输层（TCP/UDP）**和**应用层（HTTP）**。| **书籍:** 《图解HTTP》, 《图解TCP/IP》 |
| **TCP 协议** | 理解 **TCP 三次握手**（建立连接）、**四次挥手**（断开连接）的机制。掌握 **可靠性**、**流量控制**和**拥塞控制**的基本原理。| **搜索关键词:** `TCP 三次握手原理`、`TCP 拥塞控制` |
| **UDP 协议** | 理解 UDP 的**无连接**、**不可靠**特性。了解其在 DNS、实时通信（WebRTC）中的应用场景。| **搜索关键词:** `TCP 和 UDP 区别` |
| **DNS 解析** | 理解 DNS 域名解析的完整流程：**递归查询、迭代查询**，以及浏览器和本地 OS 的 DNS 缓存机制。| **搜索关键词:** `DNS 解析过程 详解` |

## 阶段二：HTTP 协议与性能优化 (HTTP & Performance)

核心目标：精通现代 HTTP 协议，并掌握加速数据传输的优化技巧。

| 知识点 | 核心内容 | 推荐学习资料 |
| :--- | :--- | :--- |
| **HTTP/1.1 瓶颈** | 队头阻塞、连接复用问题，了解 Keep-Alive 的作用。| **搜索关键词:** `HTTP 1.1 队头阻塞` |
| **HTTP/2** | **多路复用**（解决队头阻塞）、**头部压缩**（HPACK）、**服务器推送**。理解其如何大幅提升性能。| **文章:** Google Developers 上关于 HTTP/2 的系列文章 <br/>**搜索关键词:** `HTTP/2 原理 多路复用` |
| **HTTP/3 (QUIC)** | 理解 HTTP/3 基于 **UDP** 的特性。掌握 QUIC 协议如何解决 TCP 固有的队头阻塞问题和连接建立延迟。| **视频:** QCon 演讲，或 Web.dev 上的 QUIC 介绍 <br/>**搜索关键词:** `HTTP/3 QUIC 详解` |
| **HTTP 缓存** | 深入掌握**强制缓存**（`Cache-Control`）和**协商缓存**（`Etag`, `Last-Modified`）。熟练运用它们进行资源版本控制和更新。| **MDN 文档:** HTTP Caching |
| **CDN (内容分发网络)** | 理解 CDN 的工作原理、选址和缓存策略。了解前端如何通过 CDN 加速静态资源加载。| **搜索关键词:** `CDN 工作原理与加速` |

## 阶段三：安全、跨域与新趋势 (Security & Advanced)

核心目标：掌握前端网络安全边界，并了解前沿的网络通信技术。

| 知识点 | 核心内容 | 推荐学习资料 |
| :--- | :--- | :--- |
| **HTTPS 与 TLS** | 理解 **HTTPS** 的本质（HTTP + TLS）。掌握 **TLS/SSL 握手过程**（加密协商、证书验证、密钥交换）以及其对性能的影响。| **搜索关键词:** `TLS/SSL 握手过程 详解` |
| **跨域请求 (CORS)** | 深入理解 **CORS** 机制：**简单请求**和**预检请求（Preflight）**的触发条件和流程。了解如何通过 Nginx 代理、`document.domain`、`postMessage` 等方式解决跨域。| **MDN 文档:** HTTP access control (CORS) |
| **WebSocket** | 理解其在 HTTP 协议基础上建立的**双向全双工**连接。掌握其与传统的 Long Polling 的区别和应用场景（实时聊天、推送）。| **MDN 文档:** WebSockets API |
| **WebRTC** | 了解 WebRTC 的基本架构和协议栈，掌握 **P2P（点对点）**通信的建立流程（STUN/TURN/Signaling Server）。| **搜索关键词:** `WebRTC 工作原理 详解` |
| **身份认证** | 深入理解 **Session/Cookie** 认证、**Token** 认证（JWT）和 **OAuth2.0/OpenID Connect** 协议，区分它们在前后端分离架构下的应用。| **搜索关键词:** `JWT OAuth2 认证区别` |
